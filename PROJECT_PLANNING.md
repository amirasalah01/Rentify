# Rentify Degree Project Planning Document

**Project Duration:** February 27, 2026 - May 2026  

#### Milestones  
1. **Research & Planning** (Feb 27 - Mar 10)  
2. **Design Phase** (Mar 11 - Mar 24)  
3. **Development Phase** (Mar 25 - Apr 30)  
4. **Testing & Refinement** (May 1 - May 15)  
5. **Final Presentation & Submission** (May 16 - May 31)

---

### Daily Tasks and Checkpoints

**Week 1 (Feb 27 - Mar 5)**  
- **Feb 27:** Kickoff project meeting; define project scope and objectives.  
- **Feb 28:** Research existing solutions in rental space; summarize findings.  
- **Mar 1:** Identify key features for Rentify; create feature list.  
- **Mar 2:** Create user personas and user journey maps.  
- **Mar 3:** Develop project timeline with deadlines.  
- **Mar 4:** Prepare a risk assessment report.  
- **Mar 5:** Review research findings with the team; finalize project plan.

**Week 2 (Mar 6 - Mar 12)**  
- **Mar 6:** Sketch wireframes for key user interfaces.  
- **Mar 7:** Conduct team feedback session on wireframes.  
- **Mar 8:** Create a prototype of user interfaces.  
- **Mar 9:** Set up version control and collaboration tools (GitHub).  
- **Mar 10:** Finalize UI designs based on feedback.

**Week 3 (Mar 13 - Mar 19)**  
- **Mar 11:** Outline architecture: frontend and backend structure.  
- **Mar 12:** Set up the development environment.  
- **Mar 13:** Start developing the database schema.  
- **Mar 14:** Implement user authentication features.  
- **Mar 15:** Build out homepage and basic navigation.  
- **Mar 16:** Incorporate mobile responsiveness.  
- **Mar 17:** Checkpoint: Review development progress.

**Week 4 (Mar 20 - Apr 2)**  
- **Mar 18:** Develop listing features for rental properties.  
- **Mar 19:** Create search and filter functionalities.  
- **Mar 20:** Integrate payment processing system.  
- **Mar 21:** Implement user dashboard.  
- **Mar 22:** Set up notification system.  
- **Mar 23:** Review code quality and refactor where necessary.  
- **Mar 24:** Checkpoint: Feature demo to stakeholders.

**Week 5-7 (Apr 3 - Apr 30)**  
- **Apr 1:** Begin integration of APIs for additional features.  
- **Apr 2-30:** Daily development sprints on various features (assign specific tasks per day).  
- **Checkpoint (April 15):** Mid-development review; adjust timelines if necessary.  
- **April 30:** All core features should be implemented by this date.

**Testing & Refinement**  
- **May 1-10:** Conduct user testing sessions.  
- **May 11-15:** Iterate based on user feedback; bug fixing.

**Final Preparations**  
- **May 16:** Prepare presentation materials.  
- **May 17-20:** Mock presentations with peers for feedback.  
- **May 21-30:** Finalize all documentation and submit the project.  
- **May 31:** Final project deadline.

---

### Deliverables  
1. Comprehensive research report.  
2. User personas and journey maps.  
3. Complete UI prototype.  
4. Functional application with core features.  
5. User testing report.  
6. Project presentation slides.
